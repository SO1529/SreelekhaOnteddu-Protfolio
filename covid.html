AI Resume Chatbot

Interactive Resume Assistant Built with AWS

About This Project

I created an AI-powered chatbot that lives on my resume website and answers questions about my background, skills, and experience. Instead of visitors reading through a static resume, they can simply ask questions and get instant, conversational responses.

Ask the chatbot "What are Sreelekha's Gen AI skills?" or "Tell me about her projects" and get immediate, accurate answers drawn directly from my resume.

What It Does

The chatbot serves as an interactive resume assistant that:

  • Answers questions about my education and degrees
  • Explains my technical skills in Gen AI and AWS
  • Describes my projects and achievements
  • Shares my certifications and career goals
  • Provides contact information

It's available 24/7 for recruiters and visitors worldwide.

How I Built It

AWS Services Used

I leveraged AWS cloud infrastructure to create a scalable, serverless solution:

  • AWS Lambda - Runs the chatbot logic without managing servers
  • API Gateway - Handles requests from the website to the chatbot
  • Amazon Bedrock - Powers the AI with large language models
  • S3 - Stores my resume data and website files
  • DynamoDB - Logs conversations for improvements

Development Process

1. Data Preparation: I structured my resume into a knowledge base that the AI can easily reference.

2. Prompt Engineering: I crafted specific prompts to ensure the chatbot stays professional and accurate.

3. API Integration: Connected the frontend chat widget to AWS Lambda functions.

4. Testing: Asked dozens of test questions to ensure accuracy and refine responses.

5. Deployment: Deployed everything on AWS with automatic scaling.

Challenges I Solved

Keeping responses accurate: I implemented strict controls to ensure the chatbot only shares verified information from my resume—no hallucinations or made-up details.

Making it fast: Optimized the Lambda functions and prompts so responses come back in under 2 seconds.

Managing costs: Used serverless architecture to keep AWS costs minimal while maintaining performance.
